MySQL主从复制之数据库配置

数据库版本：MySQL 8.0.23
Master IP: 172.17.0.2
Slaver IP: 172.17.0.3

主数据库配置

1. 找到配置文件

文件加载优先级从左到右 mysql --help | grep 'my.cnf'

2. 修改my.cnf文件

3. 建立数据库，建立同步的用户

4. 查看状态

从数据库配置

1. 配置my.cnf文件

与master服务不同的是，slave服务的server-id不一样，复制的数据库配置也不一样

2. 建立与主库相同的数据库

3.与主服务建立连接

master_host: 主服务器
master_user：用户名
master_password：密码
master_log_file：主服务器日志文件名
master_log_pos：主服务器日志位置

slave服务读取主机上某一位置的日志文件，此后数据库操作与master进行同步

4. 启动数据库主从服务

  1. start slave
  2. show slave status\G;

查看主从配置是否存在问题

遭遇问题

  1. Slave_IO_Running的值显示为NO，同步无法进行
    报错显示两个数据库UUID一致，因为是镜像拷贝，UUID是一样的，改掉一个即可
    usr/bin/mysql 是指：mysql的运行路径
    var/lib/mysql 是指：mysql数据库文件的存放路径
    usr/lib/mysql 是指：mysql的安装路径
    修改 MySQL数据库文件中的auto.cnf文件，重启数据库后IO同步正常

常用命令

  • show master status\G;
  • show slave status\G;
  • slave stop 或者是stop slave
  • slave start 或者是start slave
  • show variables like ‘%server_uuid%’;
