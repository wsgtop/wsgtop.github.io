window中docker中使用centos

一、下载安装CentOS

  1. 进入centos可用镜像页面，查看可用的镜像centos镜像地址

    根据镜像的提示命令进行镜像下载

  2. 相关命令
    拉取指定版本的镜像：docker pull centos:centos7

    启动并运行容器：docker run -itd –name centos-test centos:centos7

    进入容器内部：docker exec -it centos-test bin/bash

    启动容器：docker start centos-test

    停止容器：docker stop centos-test

    删除容器：docker rm -f centos-test

二、centos7容器的使用

  1. 主机名

  1. 密码
    重置root用户密码：passwd root

    增加user用户：useradd user

    重置user用户密码：passwd user

  2. 查看IP

    安装net-tools: yum install net-tools -y

    查看IP：ifconfig

  3. 开启SSH服务

    (1). 安装ssh

    yum install openssh-server -y

    (2). 查看安装状态

    rpm -qa | grep openssh


    (3). 启动服务出现问题，Failed to get D-Bus connection: Operation not permitted

    经过百度，发现需要在创建容器的时候增加权限，并且不能使用默认的bash

    命令： docker run -d -name centos7 --privileged=true centos:7 /usr/sbin/init

    好家伙，换个contos8试下，据说这个问题只在centos7的某个版本才会出现

三、centos 8

(1). 选择镜像下载：docker pull centos:8

(2). 启动容器： docker run -itd -p 50001:22 –name centos8-test 8

(3). 进入容器内部：docker exec -it centos8-test bin/bash或者 docker attach centos8-test

(4). 设置密码：执行命令passwd重置root用户密码

(5). 测试命令systemctl


centos8也出现了相同的问题,咔嚓，这个还避不过去了，百度下看看，看到这个文章CSDN

还是使用centos7

命令： docker run -d -name centos7 --privileged=true centos:7 /usr/sbin/init

在window 上执行命令应该改为docker run -itd --name centos7 -p 50001:22 --privileged=true centos:centos7 init



进入容器：docker exec -it centos7 bin/bash

(6). 运行命令service service: command not found 错误

查看可安装的软件包：yum list | grep initscripts

安装：yum install initscripts -y

此时service命令可以使用,好像这一步没啥作用

(7). whereis sshd查找命令的绝对路径/usr/sbin/sshd执行启动SSH服务

报错，因为缺少公钥和私钥


运行命令生成对应的key

ssh-keygen -t rsa -b 2048 -f /etc/ssh/ssh_host_rsa_key

ssh-keygen -t ecdsa -b 256 -f /etc/ssh/ssh_host_ecdsa_key

ssh-keygen -t ed25519 -b 256 -f /etc/ssh/ssh_host_ed25519_key

此时启动ssh服务，启动成功/usr/sbin/sshd

(8). 安装telnet yum install telnet -y

测试端口 telnet 127.0.0.1 22 连接成功，centos服务器上的sshd服务已经启动成功，并且可以访问

(9). SSH连接

在启动容器的时候端口映射到本地端口50001

修改ssh服务配置，打开文件/etc/ssh/sshd_config，取消注释，允许密码登录


ssh连接：ssh root@127.0.0.1 -p 50001，确认后输入密码，连接成功
