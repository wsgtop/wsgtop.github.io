koa-body的使用

一、为什么使用

(1)使用node.js原生写法带来的问题
使用原生的对数据进行解析，操作稍显复杂，引入koa-body，简化代码，使得逻辑清楚明了

(2)使用其它包带来的问题
处理post请求可以使用koa-bodyparser
处理上传的图片使用koa-multer
koa-multer和koa-route（不是koa-router）存在不兼容问题
使用koa-body可以替换掉koa-bodyparser和koa-multer两个

其它属性

koa-body的可用选项：

参数名 描述 类型 默认值
patchKoa 将请求体打到koa的ctx.request中 Boolean true
urlencoded 是否支持urlencoded 的表单 Boolean true
text 是否解析text/plain 的表单 Boolean true
json 是否解析json请求体 Boolean true
jsonStrict 是否使用json严格模式,true 会只处理数组和对象 Boolean true
jsonLimit JSON数据体的大小限制 String/Integer 1mb
formLimit 限制表单请求体的大小 String/Integer 56kb
textLimit 限制text body 的大小 String/Integer 56kb
encoding 表单的默认编码 String utf-8
formidable 配置更多的关于multipart的选项 Object {}
onError 错误处理 Function function(){}
multipart 是否支持 multipart-formdate 的表单 Boolean false
patchNode 将请求体打到原生node.js的ctx.req中 Boolean false

formidable完整选项

参数名 描述 类型 默认值
multipart 支持多文件上传 Boolean true
maxFields 限制查询字段的数量 Integer 1000
maxFieldsSize 限制所有字段的内存量 Integer 210241024
uploadDir 设置文件上传的目录 String os.tmpDir()
onFileBegin 文件上传前的一些设置操作 Function function(name,file){}
keepExtensions 保留原始文件后缀名 Boolean false
hash 如果要计算传入文件的hash，能选择“md5”或者“sha1” String false
